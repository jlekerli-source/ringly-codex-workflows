# CI Gate Mode

CI Gate Mode runs Autopsy, generates SARIF, creates a review comment, creates badge JSON, writes step-summary Markdown, writes `gate.json`, and uploads the whole bundle when used through the composite action. The reusable action also adds a GitHub Check Run API payload to the uploaded artifact and can post it when explicitly enabled.

## Command

```bash
./bin/codex-maintainer ci-gate \
  --run fixtures/autopsy/good-run/run.md \
  --task fixtures/autopsy/good-run/task.md \
  --diff fixtures/autopsy/good-run/diff.patch \
  --tests fixtures/autopsy/good-run/tests.log \
  --policy templates/policy/default.conf \
  --out /tmp/codex-gate \
  --mode warn
```

Outputs:

- `autopsy/report.md`
- `autopsy/report.json`
- `sarif/results.sarif`
- `review/comment.md`
- `review/badge.json`
- `summary.md`
- `gate.json`
- `check-run/payload.json` when generated by the reusable action
- `check-run/response.json` when optional Check Run posting is enabled

`--mode warn` exits zero. `--mode fail` exits non-zero when the gate status is `blocked`.

## GitHub Action

```yaml
permissions:
  contents: read
  checks: write

- name: Run Codex maintainer CI gate
  uses: jlekerli-source/ringly-codex-workflows/actions/ci-gate@v3.39.0
  with:
    run: run.md
    diff: change.patch
    tests: test.log
    policy: .codex-maintainer/policy.conf
    mode: warn
    post-check-run: "true"
    github-token: ${{ secrets.GITHUB_TOKEN }}
```

The action appends `summary.md` to the GitHub Actions step summary, writes `check-run/payload.json`, optionally posts `check-run/response.json`, and uploads the configured output directory as an artifact. Leave `post-check-run` unset to keep posting disabled while still collecting the payload artifact.
